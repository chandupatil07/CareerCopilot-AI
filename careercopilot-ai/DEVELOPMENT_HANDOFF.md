## 2. Authentication System Features
- **Bcrypt CryptContext:** Uses Passlib context hashing to secure stored user passwords.
- **JWT Tokens:** Generates short-lived (15m) access tokens returned in JSON payloads, and long-lived (7d) refresh tokens saved inside HttpOnly cookies.
- **Protected Endpoints:** Built dependency injector `get_current_user` to secure routes (like `/users/me`) and verify session tokens.
- **Error Exceptions:** Standardized exceptions for duplicate emails, invalid passwords, expired tokens, and unauthorized access.

## 3. Database Migration Workflow (Alembic)
- **Autogenerate Revision:** `python -m alembic revision --autogenerate -m "Initial schema"`
- **Upgrade Database:** `python -m alembic upgrade head`
- **Rollback Migration:** `python -m alembic downgrade -1`

## 4. Technical Interview Prep Q&A

### Q1: Why is storing refresh tokens in HttpOnly cookies secure?
**Answer:** `HttpOnly` cookies cannot be accessed or read by client-side JavaScript. This prevents malicious scripts (XSS attacks) from stealing the token, securing the session from session hijacking.

### Q2: What is the purpose of salting in password hashing?
**Answer:** Salting appends random characters to a password before hashing it. This ensures that identical passwords generate unique hashes, preventing attackers from using precomputed rainbow tables to reverse-engineer passwords.
