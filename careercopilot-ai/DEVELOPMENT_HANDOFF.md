- **`security_design.md`:** Covers the JWT authentication strategy, work-factor 12 bcrypt hashing, CORS rules, and rate-limiting limits.

- **`scalability_plan.md`:** Scaling path from 10 to 1M users using load balancers, replicas, Celery task queues, and PgBouncer.
- **`error_catalog.md`:** Application-specific error codes catalog (`AUTH_001`, `RESUME_002`, `APP_001`).

## 3. Database Migration Workflow (Alembic)
- **Autogenerate Revision:** `python -m alembic revision --autogenerate -m "Initial schema"`
- **Upgrade Database:** `python -m alembic upgrade head`
- **Rollback Migration:** `python -m alembic downgrade -1`

## 4. Technical Interview Prep Q&A

### Q1: What is the advantage of using path-based API versioning over header-based versioning?
**Answer:** Path-based versioning (e.g. `/api/v1/...`) is statically analyzable, making it highly transparent and easily cached by CDNs and proxy servers. It simplifies request routing in API gateways (like Kong or NGINX) compared to parsing HTTP headers.

### Q2: Why do we separate the business logic into a Service Layer?
**Answer:** It enforces the Single Responsibility Principle. API routers only handle transport details (request parsing, status codes) and repositories only handle SQL querying. The Service Layer encapsulates the core business rules, keeping the codebase testable and preventing code duplication.

## 5. Development Commands

### Frontend Workspace
- Start dev server: `npm run dev`
- Build assets: `npm run build`

### Backend Workspace & Migrations
- Activate Virtual Env (Windows): `.\venv\Scripts\Activate.ps1`
- Install dependencies: `pip install -r requirements.txt`
- Auto-generate database migration script: `python -m alembic revision --autogenerate -m "Initial schema"`
- Run database migrations: `python -m alembic upgrade head`
- Launch ASGI API server: `uvicorn app.main:app --reload`
